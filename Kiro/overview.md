# 発表概要

当資料は社内発表用プレゼンテーション資料を作成するための基本情報や背景をまとめたものである。

## 発表基本情報

* **タイトル:** AWSの新しいIDE Kiroを”がっつり”触ってみた — 生成AI主導の仕様駆動開発でスマホアプリを動かすまでの体験共有 —
* **発表者:** SFM1部 / 大坪　範鑑 / ITアーキテクト / 麻雀愛好家
* **時間:** 発表45分 ＋ 質疑応答10分
  * 第1章（5分）：はじめに
  * 第2章（10分）：背景とKiroの概念
  * 第3章（15分）：デモ
  * 第4章（15分）：終わりに・質疑応答
* **ターゲット:** 全社（エンジニア・非エンジニア混在）。エンジニア層はCopilot/Q Developer経験者が中心。
* **発表のゴール:**
  1. 「Kiroはただの新しいIDEではない（仕様駆動である）」ことを伝える。
  2. 生成AI主導の開発体験（メリットとリアリティ）を共有する。
  3. 聴衆に「自分も触ってみたい」「ビジネスのヒントになりそう」と思わせる。

## 自身の背景

### 初期：ツールを触り始める　2025年3月頃
- Github Copilotで生成AI開発に触れる
- コード補完やチャット形式にとどまる

まだ実用的なレベルとは感じられず、チャットツールで賢いモデルを使えばいいと感じた

### 中期：生成AIでコードを修正する 2025年5月頃
- Q Developer CLIを使う
  - MCPを立ち上げて連携できる点がよい
  - プロジェクトソースを読み込んで理解した上でソースを作成できる
  - 編集内容を確認しながら承認ベースで進む
- Backlog MCPを使ってチケット駆動開発を実施
  - 開始時に計画を立ててチケット作成
  - 作業中の記録や実行ログをコメント登録
  - コミットとチケット番号の紐づけ

開発体験としてはかなり向上し、一人作業としては満足していた。
作業が継続していく中で、チャットセッションが前日の作業を覚えていないため、前日の作業（既存のコード）と整合性のないコードを生成し始める点に困った。
前回の作業チケットを最初に読ませるなどの工夫を行っていたが、前提条件が増えるにしたがって限界が見えてきた。

### 後期：Kiroを使う 2025年8月頃
- Kiroを使い始める
  - 仕様駆動開発というコンセプトがこれまでの開発の弱点を解決する手法になるのではないかと感じた
  - コミュニティも盛り上がっており、発表当初はダウンロード前にWaitlistに登録が必要になっていて、期待感が上がった
  - 使ってみて、当然課題感はあるものの、おおむね満足の開発体験を得ることができたので周りに情報共有したいと感じた

## 発表に至るまでの経緯
- 成果発表はしたいとかんじていたが、どのレベルで誰に向けて発表するか悩んでいた。
- 部会発表では対象となる技術者の数が限られるため、社内など対象を広げる必要があると考えた
- 社内では今期から生成AIを扱うチームが作られ事務局もできた。事務局メンバが知り合いだったので、発表の打診を行い、社内発表として進める形になった。

## 発表に関する調整
- 全社員は700人ほど。技術者を中心に誰が参加してもいいとアナウンスするが、100名参加してもらえれば結構集まったな、というスケール感
- 昼の休憩時間を使うため、固い発表ではなくご飯を食べながら聞くぐらいのイメージ
- 内容は個人で作業したものが全てなので社外秘情報等はない
- 事後にアンケートを考えている

## プレゼン資料の作り方
- 当プロジェクトで生成AIと相談しながらmarkdown形式のドキュメントを作成する
- NotebookLMに資料を渡して、プレゼン資料を生成する
- 発表の一部は動画を作成する
  - 経験はないが、クリップボードで動きだけ作成して、clipChampで編集したり音声を付けたりしたい（できるかどうかは知らない）

### 構成
- overview.md: 発表の骨子や背景を説明する前提ドキュメント
- presentation.md: 発表スライドと対となるドキュメント。各ページに記載すべき内容を示す
- screenshots: スライドの一部として使用するスクリーンショット
- movies: 動画ファイル
